# Backtest Engine

The first QuantLab backtest engine consumes target portfolio weights and daily
adjusted-close returns.

## Inputs

- Market panel with `date`, `ticker`, `adj_close`.
- Target weights with `date`, `ticker`, `weight`.
- Optional `sector` metadata for holdings and sector exposure reporting.

Target weights are generated by the strategy layer. The backtest layer does not
care whether the signal came from a factor, an ML model, or a Transformer.

## Current Accounting

1. On a rebalance date, move from current weights to target weights before that
   date's asset return is applied.
2. Compute turnover as the absolute weight change.
3. Deduct commission plus slippage from NAV.
4. Apply daily weighted asset returns.
5. Credit interest on `1 - net_exposure` cash weight and charge borrow fees on
   absolute short exposure.
6. Record NAV, turnover, transaction costs, financing, and holdings.
7. When sector metadata is available, record sector-level long, short, gross,
   and net exposure.

This is a research backtester, not a broker-grade execution simulator. Cash and
borrow rates use simple annual-rate-to-daily conversion; broker-specific margin,
locate availability, and tiered borrow schedules are outside the current scope.

## Realism Controls

- `execution_lag_days`: shifts target weights forward by N trading days.
- `min_dollar_volume`: removes positions below a rolling average dollar-volume
  threshold.
- `liquidity_window`: lookback window used for average dollar volume.
- `benchmark_ticker`: optional ticker from the market panel used for
  benchmark-relative statistics. A missing configured ticker fails fast.
- `annual_cash_rate`: annualized rate credited to cash weight. A negative cash
  weight represents financed net-long exposure at the same configured rate.
- `annual_borrow_rate`: annualized stock-borrow fee charged on absolute short
  exposure.

## Metrics

- Total Return
- Annual Return / CAGR
- Annual Volatility
- Sharpe Ratio
- Sortino Ratio
- Max Drawdown
- Calmar Ratio
- Win Rate
- Historical 95% Value at Risk (VaR)
- Historical 95% Conditional Value at Risk (CVaR / Expected Shortfall)
- Profit Factor
- Average Turnover
- Total Cost
- Total Cash Interest
- Total Borrow Cost

When `benchmark_ticker` is configured, `metrics.json` also contains:

- Tracking Error
- Information Ratio
- Market Beta
- Annualized Jensen-style Alpha

## Workflow

```powershell
python scripts/run_backtest.py --config configs/backtest.yaml
```

Outputs:

```text
artifacts/backtests/
  equity_curve.csv
  holdings.parquet
  trade_log.parquet
  exposure.csv
  sector_exposure.csv
  metrics.json
```
